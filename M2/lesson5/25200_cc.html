There are five categories of environments which factor into Critical Technology Element identification. These categories relate to both internal and external environments of the system. The categories are physical, logical, data, security, and user and use.

Select each of the environment categories to view additional information, and then select the TRA in Action icon to view the next segment of the TRA process scenario.

define the environment of the Critical Technology Element. When doing so, you will need to ask some questions about the technology element.
Key questions that apply to all environments include:

  1. Is the physical / logical / data / security environment in which this Critical Technology Element has been demonstrated similar to the intended environment? If not, how is it different?
  2. Is the Critical Technology Element going to be operating at or outside its usual performance envelope?
  3. Do the design specifications address the behavior of the Critical Technology Element under these conditions?
  4. What is unique or different about this proposed operations environment?
  5. Do test data, reports, or analyses that compare the demonstrated environment to the intended environment exist?
  6. If modeling and simulation are important aspects of that comparison, are the analysis techniques common and generally accepted?
